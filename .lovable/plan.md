
## Төлөвлөгөө — Санхүү дээр ашиглалтын дүн автоматаар дуудагдах алдааг засах

### Асуудлын шалтгаан
`src/pages/Finance.tsx` дээр нэхэмжлэх үүсгэх dialog-ийн автомат бөглөлт одоогоор `handleSelectTenant()` дотор хийгдэж байна. Энэ үед:

1. **Хугацаа (`newPeriod`) default-оор одоогийн сар** (`new Date().toISOString().slice(0, 7)`) авч байгаа тул meter store-д байхгүй сар дээр 0₮ болж байна.
2. **Хугацаа солих үед** `setNewPeriod()` дараа шууд `handleSelectTenant(newTenant)` дуудагдаж байгаа ч React state хараахан шинэчлэгдээгүй тул **хуучин period**-оор ашиглалтын дүн бодогдож байна.
3. Иймээс ашиглалтын зөрүү бодогдсон ч нэхэмжлэх үүсгэх хэсэгт тогтвортой орж ирэхгүй байна.

### Хийх засвар

#### 1. Автомат бөглөлтийг state-ээс хамааралтай болгох
`src/pages/Finance.tsx`
- `handleSelectTenant()`-ийг зөвхөн tenant сонгох handler биш, харин:
  - `newTenant`
  - `newPeriod`
  - `meterStore`
- эдгээр өөрчлөгдөх бүрт дахин тооцоолдог тусдаа autofill функц / `useEffect` болгон салгана.

Ингэснээр:
- түрээслэгчээ эхэлж сонгосон ч,
- хугацаагаа дараа нь сольсон ч,
- ашиглалтын бүртгэл дараа нь хадгалагдсан ч,
санхүүгийн dialog доторх ашиглалтын төлбөр зөв шинэчлэгдэнэ.

#### 2. Period-ийн default утгыг өгөгдөлтэй логик болгох
`src/pages/Finance.tsx`
- `newPeriod`-ийн default-ийг одоогийн бодит огноогоор биш,
  - meter store-д байгаа хамгийн сүүлийн сар,
  - эсвэл UI-д ашиглаж буй demo сарын хүрээтэй уялдуулж
тохируулна.

Ингэснээр dialog нээхэд шууд 0₮ биш, бодит demo өгөгдөлтэй сар автоматаар сонгогдоно.

#### 3. Ашиглалтын дүн тооцооллын хамгаалалт сайжруулах
`src/pages/Finance.tsx`
- `meterStore[newPeriod]?.[tenant.id]`
- `meterStore[previousMonth(newPeriod)]?.[tenant.id]`
утгуудыг шалгаж:
  - тухайн сарын уншилт байвал зөрүүгээр,
  - өмнөх сар байхгүй бол одоогийн уншилтаар,
  - хоёулаа байхгүй бол 0₮
гэсэн логикийг тодорхой, тогтвортой хэвээр хадгална.
- `autoFillNote` дээр яг ямар нөхцөлөөр дүн бөглөгдсөнийг илүү ойлгомжтой харуулна.

#### 4. Нэхэмжлэх үүсгэх dialog-ийн урсгалыг тогтвортой болгох
`src/pages/Finance.tsx`
- Түрээслэгч сонгох, хугацаа сонгох хоёрын дараалал хамаарахгүй ажиллахаар засна.
- Dialog дахин нээгдэхэд өмнөх үлдэгдэл state-үүдээс болж буруу дүн харагдахгүйгээр reset логик нэмнэ.
- Ашиглалтын төлбөр автоматаар орж ирсэн эсэхийг UI дээр илүү тодорхой болгоно.

#### 5. Давхар жижиг засвар
`src/pages/Finance.tsx`
- Console дээр харагдаж буй `Dialog` accessibility warning-ийг арилгахын тулд `DialogDescription` нэмнэ.
- `DialogFooter` орчимд гарч буй ref warning-ийг шалгаж, function component-д ref дамжиж байгаа хэсгийг шейдсэн компонентын хэв маягт нийцүүлэн цэвэрлэнэ.

### Хүлээгдэх үр дүн
Засварын дараа:
- Ашиглалт хэсэгт хадгалсан сарын дүн Санхүү дээр нэхэмжлэх үүсгэх үед автоматаар орж ирнэ
- Түрээслэгч эсвэл хугацаа аль нэгийг нь солиход ашиглалтын төлбөр шууд дахин бодогдоно
- Өмнөх сарын зөрүүгээр тооцсон дүн нэхэмжлэхийн `Ашиглалтын төлбөр` талбарт зөв fill хийгдэнэ
- Dialog warnings багасна
