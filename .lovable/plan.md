

## Төлөвлөгөө

Хэрэглэгчийн хүсэлтийг 4 үндсэн хэсэгт хуваан хэрэгжүүлнэ.

### 1. Түрээслэгчийн хөрөнгө сонголт дотор хайлт нэмэх
`src/pages/Tenants.tsx` файлын property multi-select хэсэгт хайлтын `Input` нэмж, объектын нэр, талбайн дугаар, давхараар шүүх боломжтой болгоно.

### 2. Нэвтрэх / Бүртгүүлэх систем
- **`src/contexts/AuthContext.tsx`** — Client-side auth context үүсгэнэ. localStorage-д хэрэглэгчдийг хадгална. Анхны admin хэрэглэгч: `admin` / `pass123$` (и-мэйл + нууц үг).
- **`src/pages/Login.tsx`** — Нэвтрэх болон бүртгүүлэх форм (и-мэйл, нууц үг). Tab-аар "Нэвтрэх" / "Бүртгүүлэх" сольно.
- **`src/App.tsx`** — AuthProvider-ээр бүхнийг боож, нэвтрээгүй бол Login хуудас руу чиглүүлнэ.
- **Sidebar** — "Гарах" товчийг logout функцтай холбоно.

### 3. Санхүү — Илгээсэн нэхэмжлэлийн жагсаалт
`src/pages/Finance.tsx` файлд шинэ Collapsible section нэмж, илгээгдсэн нэхэмжлэлүүдийг (дугаар, түрээслэгч, дүн, огноо, төлөв) хүснэгт хэлбэрээр харуулна.

### 4. Санхүү — Авлага, өглөгийн бүртгэлийг жагсаалтаар харуулах
Одоо байгаа receivables болон payables Collapsible хэсгүүдийг анхнаасаа нээлттэй болгож, жагсаалт хэлбэрээр (хүснэгт) илүү дэлгэрэнгүй харуулна.

### Засварлах файлууд
| Файл | Өөрчлөлт |
|---|---|
| `src/contexts/AuthContext.tsx` | Шинэ — auth context, admin хэрэглэгч |
| `src/pages/Login.tsx` | Шинэ — нэвтрэх/бүртгүүлэх хуудас |
| `src/App.tsx` | AuthProvider, protected routes |
| `src/components/AppSidebar.tsx` | Logout холбох |
| `src/pages/Tenants.tsx` | Property selector дотор хайлт |
| `src/pages/Finance.tsx` | Илгээсэн нэхэмжлэл жагсаалт, авлага/өглөг нээлттэй |

**Анхааруулга:** Нэвтрэлтийн систем нь client-side (localStorage) дээр суурилсан бөгөөд бодит аюулгүй байдлын шаардлага хангахгүй. Цаашид Supabase Auth руу шилжүүлэх боломжтой.
